# Plan: Fix SHAP Feature Importance Sync and Rendering

## Problem Description
1. The SHAP feature importance plot on the **Leaderboard** screen displays fallback dummy values (e.g. `petal_length`, `petal_width` from Iris dataset) because `shapData` is evaluated as `null`.
2. This happens because `/api/runs/{session_id}/shap` returns `"status": "pending"` and `"features": []` for the winning model (e.g., `GradientBoostingClassifier` in the current session) if it was skipped or has no SHAP CSV file on disk.
3. The frontend bounded poll (`useBoundedPoll`) immediately stops when `leaderboard.status === 'complete'`, causing the UI to never poll again and get stuck with the dummy values.
4. When transitioning from the Training page to the Leaderboard via the "Continue to Leaderboard" button, we need the SHAP values of the winning model (or the best top-1 model that actually has SHAP analysis) to auto-sync.

## Plan details

### 1. Backend Changes (evaluation.py)
* Update `_default_shap_model()` in `backend/routers/evaluation.py` to:
  1. Check if `judge_decision["selected_model"]` has a SHAP CSV file on disk.
  2. If not, check the models in `judge_decision["selected_models"]` in order.
  3. If not, check the models in `judge_decision["ranked_models"]` in order of rank.
  4. Finally, fall back to any model with a SHAP CSV on disk.
* Update `shap_importance()` to check `self.shap_status().get("status")`. If it is `"completed"` or `"failed"`, return `"status": "complete"` instead of `"pending"` even if the resolved model has no SHAP CSV on disk.

### 2. Frontend Changes (LeaderboardScreen.jsx & TrainingPage.jsx)
* Update `pollLeaderboard` in `LeaderboardScreen.jsx` to return the complete response object containing `{ leaderboard, shap, verdict, tokens }`.
* Update `useBoundedPoll` options for `pollLeaderboard` to ensure the poll continues until all critical post-training stages (`leaderboard`, `shap`, `verdict`, and `tokens`) are complete.
* Track `shapModelName` state in both `LeaderboardScreen.jsx` and `TrainingPage.jsx`.
* Render the dynamic model name in the SHAP panel headers: `SHAP Feature Importance (ModelName)`.

### 3. Verification & Testing
* Start/restart the backend and frontend to verify the API returns the correct fallback model and complete status.
* Check the Leaderboard screen to verify the real SHAP values are fetched and rendered without junk values.
