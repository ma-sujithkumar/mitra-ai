2. Resolved Ambiguities
The following questions were left open in v1 or in the meeting transcript. Each is closed with a concrete
decision.
Ambiguity

Decision

How does the USL Agent know which
columns are features if there is no output
column?

metadata.json always includes output_cols: [] for unsupervised
problems. The agent uses all non-dropped columns as features.
If output_cols is non-empty, USL is never chosen.

Which port does bin/mitra expose to the
host?

Streamlit: 8501, FastAPI: 8001, Ray head: 6379, Ray
dashboard: 8265. All four are EXPOSE'd in the Dockerfile. Only
8501 and 8001 are mapped by default in docker run examples.

What happens when two models finish
training concurrently on Ray?

Each model_NNN/ subdirectory is created atomically before
ray.remote() is called. Writers never conflict. The leaderboard is
re-sorted on every GET /leaderboard call by reading all
metrics.json files.

Does the Error/Debug Agent have a loop
limit?

Yes. Max 3 self-correction attempts per script. On the 4th failure,
verdict = abort and the model is skipped. The limit is stored in
session config, not hard-coded, so it is tunable.

How does the Feature Selection Agent
avoid writing features that do not exist?

It outputs a feature_selection.json with keep and drop arrays that
are validated against data_encoded.csv column headers before
the downstream scaler runs. Mismatch = validation error + abort.

What if the LLM returns model names that
have no template?

TemplateResolver returns a fallback flag. The family agent then
invokes the write-from-scratch path with the LLM, using the
closest template as a few-shot example in the prompt.

How does Streamlit P2 receive SSE events
without a full page refresh?

Streamlit does not support native SSE. P2 uses st.empty() + a
Python requests.get(..., stream=True) loop in a st.fragment with
a 500ms re-run interval. Each event appends to a session_state
log list.

What is mini_data.csv exactly?

pandas.describe(include='all') + dtype per column + null_count +
unique_count + is_pii_suspect (bool, from column-name
heuristic). Capped at 1000 rows for sampling. Always generated
before any LLM call.

5. SSE Event Bus — Low-Level Design
▲ Improvement from v1: v1 said 'SSE feed' with no design. This section gives the complete
implementation: queue shape, FastAPI generator, Streamlit consumer, and agent-side tool signature.

Figure 4 — SSE event flow: agent tool → asyncio queue → FastAPI generator → Streamlit polling loop.

5.1 Event Schema
{"session_id":"", "stage":"feature_selection",
"level":"info"|"warn"|"error",
"msg":"chi-square on 12 cols",
"pct":42, // 0-100 within this stage
"ts":"2026-05-24T10:32:11Z"}

5.2 FastAPI SSE Endpoint
async def event_stream(session_id: str):
q = session_queues[session_id] # asyncio.Queue, created at POST /session
while True:
event = await asyncio.wait_for(q.get(), timeout=300)
if event is SENTINEL: break
yield f"data: {json.dumps(event)}\n\n"
@app.get("/session/{sid}/events")
async def stream(sid: str):
return StreamingResponse(event_stream(sid),
media_type="text/event-stream")

5.3 emit_event() — ADK Tool (agent side)
def emit_event(stage:str, msg:str, pct:int=0, level:str="info") -> str:
"""ADK FunctionTool that any agent can call to push a status update."""
event = {"session_id": ctx.session_id, "stage": stage, "level": level,
"msg": msg, "pct": pct, "ts": utcnow()}
asyncio.get_event_loop().call_soon_threadsafe(q.put_nowait, event)
return "ok"

11.1 Template Resolution Protocol (all family agents)
•

Step 1: Read model_config.json[i].family.

•

Step 2: Look up mitra/templates/{family}/train.py.jinja2. If exists → go to Step 3.
Else → Step 5.

•

Step 3: Read hp_space.yaml from same folder. Merge with model_config[i].hp_space (config
overrides template defaults).

•

Step 4: Render Jinja2 template with {n_classes, input_dim, output_cols, hp_dict}. Write
train_script.py. Done.

•

Step 5 (fallback): Call LLM with the closest template as a few-shot example. LLM writes the full
script. Run syntax check (py_compile). If fails → retry once.
■ Risk addressed: R3 — Template regen: Step 2 means the LLM is only called for genuinely novel
architectures. All standard models (XGBoost, RF, MLP, CNN) will always hit Step 4.

14. Low-Level Design: HPT Agent
Owner

Type

Tools

P7

Python (Optuna, no
LLM in loop)

optuna_run, ray_submit

Inputs
•

model_config.json[i]

•

hp_space.yaml (from template folder)

•

verdict.json (with next_hp_hint)

Outputs
•

new_hp.json: {trial_id, params:{}, expected_improvement}

•

hp_history.json: all trial results appended (for P3 visualization)

Key guardrails
•

No LLM in the loop — Optuna runs pure Python. LLM cost per trial would be prohibitive.

•

next_hp_hint from Judge Agent biases the Optuna sampler: suggested 'reduce' params are capped
at their current value, 'increase' params floored.

•

Hard budget: max 20 trials OR 5 minutes wall-clock per model (whichever comes first). Enforced via
Optuna callbacks.

•

Stratified sampling for class-imbalanced datasets (split preserves class ratios).

•

Each trial runs as ray.remote() so the HPT agent process is never blocked.

Prompt skeleton (agents/hpt_agent/prompt.md)
# Not a prompt — Python implementation: def objective(trial): hp =
sample_from_space(trial, hp_space, hint=verdict.next_hp_hint) ref =
ray_submit(train_script, hp, data_path) metrics = ray.get(ref) return metrics['val_score']
study = optuna.create_study(direction='maximize',
sampler=optuna.samplers.TPESampler(seed=42)) study.optimize(objective, n_trials=20,
timeout=300, callbacks=[MaxRetriesCallback(3)])

16. Low-Level Design: Ray Wrapper
The Ray Wrapper is a deterministic Python module, not an agent. It has four responsibilities, each with
a clear failure mode.
•

Start: ray.init(address='auto', ignore_reinit_error=True). If Ray is not running,
spawns a local head with ray.init(num_cpus=os.cpu_count()-1). Reports to /health
endpoint.

•

Submit: @ray.remote(num_cpus=N, num_gpus=G) decorated run() function. N and G come
from resources.yaml. Falls back to CPU-only if no GPU found.

•

Stream logs: Ray's ray.get() is blocking; logs are tailed via a background thread that reads the
Ray worker's stdout and emits SSE events every 2 seconds.

•

Teardown: On DELETE /session, ray.cancel(ref, force=True) is called for any active refs
tied to that session. Ray cluster stays alive for the next session.
■ Risk addressed: R5 — Ray fails to start: /health checks ray_ok on startup. P1 blocks session
creation if ray_ok=false, showing 'Ray not ready — check Docker resource limits'.
