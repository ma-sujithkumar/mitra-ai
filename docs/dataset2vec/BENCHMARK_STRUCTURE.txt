================================================================================
TYPICAL WORKFLOW
================================================================================

1. PREPARE DATASETS
   python benchmark/run_benchmark.py -c config/config.ini
   └── Downloads 4 UCI datasets, converts to NPZ format

2. RUN QUERIES
   └── Executes query.py on each external dataset with --verify
   └── Trains suggested models on test set
   └── Compares expected vs achieved metrics

3. ANALYZE
   └── Extracts statistics: transfer quality, deltas, neighbors
   └── Grades each dataset: EXCELLENT/GOOD/FAIR/POOR
   └── Computes summary statistics across all datasets

4. REPORT
   └── Generates human-readable reports
   └── Saves to: BENCHMARK_SUMMARY.txt + per-dataset reports
   └── Prints console summary

================================================================================
KEY METRICS
================================================================================

Transfer Quality (%)
  Definition: % of top-N models achieving within-tolerance performance
  Range: 0-100%
  Interpretation:
    80%+ = EXCELLENT (use suggestions confidently)
    50-70% = GOOD (tune the models)
    30-50% = FAIR (significant tuning)
    <30% = POOR (dataset too dissimilar)

Mean |Delta|
  Definition: Average |achieved_metric - expected_metric|
  Range: 0.0 (perfect) to 1.0 (worst)
  Interpretation:
    <0.05 = EXCELLENT (±5% error)
    0.05-0.10 = GOOD (±10% error)
    0.10-0.20 = FAIR (±20% error)
    >0.20 = POOR (>20% error)

Neighbor Similarity
  Definition: Embedding-space similarity score (inner product after L2 norm)
  Range: 0.0 (different) to 1.0 (identical)
  Typical: 0.99+ for datasets in corpus, varies for external

================================================================================
EXTERNAL DATASETS USED
================================================================================

1. car-evaluation
   Source: UCI ML Repository
   Task: Multi-class classification (4 classes: unacc, acc, good, vgood)
   Features: 6 (categorical: buying, maint, doors, persons, lug_boot, safety)
   Rows: 1,728
   Domain: Automotive
   Transfer: Usually POOR (structure differs from corpus)

2. monks
   Source: UCI ML Repository
   Task: Binary classification (synthetic benchmark)
   Features: 6 (synthetic attributes)
   Rows: 124
   Domain: Synthetic benchmark
   Transfer: Usually GOOD (but synthetic nature limits transfer)

3. spambase
   Source: UCI ML Repository
   Task: Binary spam/ham classification
   Features: 57 (word frequencies, character frequencies)
   Rows: 4,601
   Domain: Email/text
   Transfer: Usually EXCELLENT (exact or near-exact in corpus)

4. letter-recognition
   Source: UCI ML Repository
   Task: Multi-class letter recognition (A-Z, 26 classes)
   Features: 16 (morphological features)
   Rows: 20,000
   Domain: Computer vision
   Transfer: Usually EXCELLENT (exact in corpus)

# Skip preparation (datasets already prepared)
python benchmark/run_benchmark.py -c config/config.ini --skip-prepare -v

# Skip sanity check (don't test corpus dataset)
python benchmark/run_benchmark.py -c config/config.ini --skip-sanity -v

# Skip both
python benchmark/run_benchmark.py -c config/config.ini --skip-prepare --skip-sanity -v
